## 1. 🎯 Summary

- **기능명:** [UI-001] 소셜 로그인 페이지 UI — 카카오/네이버 OAuth 버튼 + 게스트 체험 + Supabase signInWithOAuth() 호출
- **목적 (Why):**
  - **비즈니스:** 서비스의 첫 진입 지점. 3040 맞벌이 부부가 출퇴근 중 모바일에서 빠르게 접근하는 소셜 로그인 경험. OAuth-only로 비밀번호 관리 부담 제거.
  - **사용자 가치:** 카카오/네이버 소셜 계정으로 원탭 로그인, OAuth 장애 시 게스트 체험 모드로 즉시 서비스 탐색.
- **범위 (What):**
  - ✅ 만드는 것: `app/login/page.tsx` (Server Component), `components/auth/LoginForm.tsx` (Client Component — signInWithOAuth 호출), 카카오/네이버 브랜드 컬러 버튼, 게스트 체험 버튼, MOCK-005 분기 로직
  - ❌ 만들지 않는 것: OAuth 백엔드 로직(CMD-AUTH-001~004 범위), 비밀번호 입력 필드(OAuth-only), 결제 관련 UI, NextAuth.js 코드
- **복잡도:** L
- **Wave:** 4 (UI 트랙)

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 카카오 로그인 → signInWithOAuth 호출
- **Given** `/login` 페이지 렌더링
- **When** "카카오로 시작하기" 버튼 클릭
- **Then** `supabase.auth.signInWithOAuth({ provider: 'kakao', options: { redirectTo: '.../auth/callback' } })` 호출, 버튼 "로그인 중..." + `disabled`

**AC-2 (정상):** 네이버 로그인 → signInWithOAuth 호출
- **Given** `/login` 페이지 렌더링
- **When** "네이버로 시작하기" 버튼 클릭
- **Then** `supabase.auth.signInWithOAuth({ provider: 'naver' })` 호출, 버튼 배경색 `#03C75A`

**AC-3 (정상):** 게스트 → /diagnosis 리다이렉트
- **Given** `/login` 페이지 렌더링
- **When** "로그인 없이 체험하기" 버튼 클릭
- **Then** CMD-AUTH-004 게스트 모드 진입 → `/diagnosis` 리다이렉트

**AC-4 (예외):** OAuth 에러 → Sentry + UI 복원
- **Given** signInWithOAuth가 에러 반환
- **When** 카카오 버튼 클릭
- **Then** Sentry 에러 전송, 버튼 다시 활성화, 에러 토스트 표시

**AC-5 (경계):** 비밀번호 입력 필드 0건
- **Given** `app/login/`, `components/auth/` 전체
- **When** `grep -r 'type="password"'` 실행
- **Then** 매칭 0건 (OAuth-only)

**AC-6 (UI/접근성):** 키보드 내비게이션
- **Given** `/login` 페이지 렌더링
- **When** Tab으로 이동, Enter로 클릭
- **Then** 카카오 → 네이버 → 게스트 순서 focus, 모든 버튼 `aria-label` 존재

**AC-7 (UI):** 모바일 반응형 (375px)
- **Given** viewport 375px
- **When** `/login` 접근
- **Then** 버튼 `w-full`, 터치 영역 ≥ 44px (`h-12` = 48px)
