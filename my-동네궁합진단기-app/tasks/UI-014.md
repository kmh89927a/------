- **기능명:** [UI-014] 이전 조건 불러오기 버튼 + 폼 자동 채움 UI — Rev 1.1 단순화

- [ ] **3.1** `components/diagnosis/LoadSavedSearchButton.tsx` — 클라이언트 컴포넌트
  ```typescript
  'use client';
  import { useState, useEffect } from 'react';
  import { Button } from '@/components/ui/button';
  import { useFormContext } from 'react-hook-form';
  import { getSavedSearch } from '@/app/actions/save-search';
  import { useToast } from '@/components/ui/use-toast';
  import * as Sentry from '@sentry/nextjs';

  export function LoadSavedSearchButton() {
    const { setValue } = useFormContext();
    const { toast } = useToast();
    const [hasSaved, setHasSaved] = useState(false);
    const [isLoading, setIsLoading] = useState(false);

    useEffect(() => {
      getSavedSearch().then(result => {
        if (result) setHasSaved(true);
      }).catch(() => {/* best effort — 실패 무시 */});
    }, []);

    async function handleLoad() {
      setIsLoading(true);
      try {
        const result = await getSavedSearch();
        if (!result) return;

        // 폼 필드 자동 채우기
        if (result.saved?.searchParams) {
          const params = result.saved.searchParams as any;
          if (params.addressA) setValue('addressA', params.addressA);
          if (params.addressB) setValue('addressB', params.addressB);
          if (params.filters) setValue('filters', params.filters);
        }

        // geocoding 실패 시 토스트
        if (result.geocodingFailures?.addressA || result.geocodingFailures?.addressB) {
          toast({
            variant: 'destructive',
            title: '저장된 주소를 불러올 수 없습니다',
            description: '주소를 다시 입력해주세요',
          });
        } else {
          toast({
            title: '이전 조건이 복원되었습니다',
            description: '필요 시 수정 후 진단을 시작하세요',
          });
        }
      } catch (e) {
        Sentry.captureException(e);
        toast({
          variant: 'destructive',
          title: '조건 불러오기 실패',
          description: '잠시 후 다시 시도해주세요',
        });
      } finally {
        setIsLoading(false);
      }
    }

    if (!hasSaved) return null;

    return (
      <Button
        onClick={handleLoad}
        variant="outline"
        size="sm"
        disabled={isLoading}
        id="btn-load-saved-search"
        aria-label="이전 진단 조건 불러오기"
      >
        {isLoading ? '불러오는 중...' : '이전 조건 불러오기'}
      </Button>
    );
  }
  ```

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 저장된 조건 있을 때만 버튼 표시
- **Given** 인증된 사용자, SavedSearch 테이블에 레코드 존재
- **When** 주소 입력 화면 로드
- **Then** "이전 조건 불러오기" 버튼 표시

**AC-2 (정상):** 클릭 → 폼 필드 자동 채우기
- **Given** 저장된 조건 `{ addressA: "강남역", addressB: "판교역", filters: { maxCommuteTime: 60 } }`
- **When** "이전 조건 불러오기" 클릭
- **Then** `setValue('addressA', '강남역')`, `setValue('addressB', '판교역')`, `setValue('filters', ...)` 호출. 폼 필드에 값 표시

**AC-3 (예외):** geocoding 실패 시 토스트 안내
- **Given** 저장된 주소A가 geocoding 실패 (주소 체계 변경)
- **When** "이전 조건 불러오기" 클릭
- **Then** "저장된 주소를 불러올 수 없습니다 — 주소를 다시 입력해주세요" destructive 토스트

**AC-4 (도메인 핵심 — Rev 1.1):** 비교 뷰 0건 정적 검증

- **Then** 매칭 0건 — Rev 1.1 단순화 핵심

**AC-5 (경계):** 저장된 조건 없을 때 버튼 숨김

## 9. 🚧 Open Questions / Risks (보류 사항)

1. **폼 채우기 후 사용자 수정 vs 자동 진단 시작:** 현재는 폼 채움 후 사용자가 수정할 기회를 제공 (자동 진단 시작 안 함). 정책 결정 필요.
2. **Toast vs 인라인 메시지:** geocoding 실패 시 Toast vs 폼 인라인 에러 중 선택. 현재 Toast + 해당 필드 비우기로 구현. 사용성 테스트 후 조정.
3. **복원 시간 ≤1초:** QRY-SAVE-001의 Prisma SELECT + geocoding 재검증 합산이 1초 이내인지. 네트워크 지연 시 초과 가능 — 버튼 로딩 상태로 대응.
4. **싱글 모드 저장 조건:** 싱글 모드(addressB 없음)에서 저장된 조건을 커플 모드 폼에 불러올 때 addressB 비어있는 경우 처리.
