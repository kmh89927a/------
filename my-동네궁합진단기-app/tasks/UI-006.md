## 1. 🎯 Summary

- **기능명:** [UI-006] 공유 링크 생성 버튼 + 클립보드 복사 확인 UI
- **목적 (Why):**
  - **비즈니스:** 진단 결과 화면에서 사용자가 원클릭으로 공유 링크를 생성하고 클립보드에 복사하여, 카카오톡 등 메신저를 통해 배우자에게 즉시 전달할 수 있게 한다. ShareLink 바이럴 funnel의 시작점이다.
  - **사용자 가치:** 복잡한 URL을 수동으로 복사할 필요 없이, 버튼 한 번으로 공유 링크가 클립보드에 복사되며 한국어 토스트 메시지로 성공/실패를 즉시 확인할 수 있다.
- **범위 (What):**
  - ✅ 만드는 것: ShareButton Client Component (`'use client'`), CMD-SHARE-001 `createShareLink` Server Action 호출, `navigator.clipboard.writeText()` 복사 + fallback (textarea + execCommand), shadcn/ui Toast 한국어 메시지, 로딩 스피너, 에러 핸들링
  - ❌ 만들지 않는 것: 공유 링크 생성 로직(CMD-SHARE-001), SSR 공유 리포트(UI-007), 전환 모달(UI-008), 결제 관련 코드, NextAuth.js 코드
- **복잡도:** L
- **Wave:** 4 (UI ShareLink 트랙)

- [ ] **3.1** `components/share/share-button.tsx`에 ShareButton Client Component 작성
  ```typescript
  'use client';
  import { useState } from 'react';
  import { Button } from '@/components/ui/button';
  import { useToast } from '@/components/ui/use-toast';
  import { Share2, Loader2, Check } from 'lucide-react';
  import { createShareLink } from '@/app/actions/share';
  import { reportErrorToSentry } from '@/lib/helpers/sentry-error';

  interface ShareButtonProps {
    diagnosisId: string;
    disabled?: boolean;
  }

  export function ShareButton({ diagnosisId, disabled }: ShareButtonProps) {
    const { toast } = useToast();
    const [isLoading, setLoading] = useState(false);
    const [isCopied, setIsCopied] = useState(false);

    async function handleShare() {
      setLoading(true);
      try {
        const { shareUrl } = await createShareLink({ diagnosisId });
        await copyToClipboard(shareUrl);
        setIsCopied(true);
        toast({
          title: '공유 링크가 복사되었습니다',
          description: shareUrl,
        });
        setTimeout(() => setIsCopied(false), 3000);
      } catch (error) {
        reportErrorToSentry(error instanceof Error ? error : new Error('공유 링크 생성 실패'));
        toast({
          variant: 'destructive',
          title: '공유 링크 생성에 실패했습니다',
          description: '잠시 후 다시 시도해 주세요.',
        });
      } finally {
        setLoading(false);
      }
    }

    return (
      <Button
        id="share-button"
        onClick={handleShare}
        disabled={isLoading || disabled}
        aria-label="진단 결과 공유 링크 생성"
        className="w-full md:w-auto"
      >
        {isLoading ? (
          <Loader2 className="mr-2 h-4 w-4 animate-spin" />
        ) : isCopied ? (
          <Check className="mr-2 h-4 w-4" />
        ) : (
          <Share2 className="mr-2 h-4 w-4" />
        )}
        {isLoading ? '링크 생성 중...' : isCopied ? '복사됨!' : '공유하기'}
      </Button>
    );
  }
  ```

- [ ] **3.2** `lib/utils/clipboard.ts`에 클립보드 복사 유틸리티 작성 (navigator.clipboard + fallback)
  ```typescript
  /**
   * 클립보드에 텍스트를 복사한다.
   * navigator.clipboard 미지원 브라우저에서는 textarea fallback을 사용한다.
   * iOS Safari에서는 사용자 제스처 컨텍스트 내에서만 동작한다.
   */
  export async function copyToClipboard(text: string): Promise<void> {
    if (navigator.clipboard && window.isSecureContext) {
      await navigator.clipboard.writeText(text);
      return;
    }

    // Fallback: textarea + execCommand
    const textArea = document.createElement('textarea');
    textArea.value = text;
    textArea.style.cssText = 'position:fixed;left:-9999px;top:-9999px;opacity:0';
    document.body.appendChild(textArea);
    textArea.focus();
    textArea.select();
    try {
      const success = document.execCommand('copy');
      if (!success) throw new Error('execCommand copy failed');
    } finally {
      document.body.removeChild(textArea);
    }
  }
  ```

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 공유 버튼 클릭 → 링크 생성 + 클립보드 복사 성공
- **Given** 진단 결과 화면에서 유효한 `diagnosisId`가 있는 상태
- **When** "공유하기" 버튼 클릭
- **Then** CMD-SHARE-001 `createShareLink` 호출 → `navigator.clipboard.writeText()` 성공 → 토스트 메시지 "공유 링크가 복사되었습니다" + 생성된 URL 표시. 버튼 아이콘이 체크마크로 3초간 전환

**AC-2 (정상):** 로딩 상태 표시
- **Given** 공유 버튼 클릭 직후
- **When** CMD-SHARE-001 응답 대기 중
- **Then** Button `disabled` + Loader2 스피너 아이콘 + "링크 생성 중..." 텍스트. 중복 클릭 차단

**AC-3 (예외):** 클립보드 권한 거부 시 fallback
- **Given** `navigator.clipboard` 사용 불가 (HTTP 환경 또는 권한 거부)
- **When** 공유 링크 생성 성공 후 클립보드 복사 시도
- **Then** textarea + execCommand fallback 실행. fallback도 실패 시 URL 직접 표시 + "아래 링크를 직접 복사해 주세요" 토스트

**AC-4 (예외):** Server Action 실패 → 에러 토스트
- **Given** CMD-SHARE-001이 에러 반환 (네트워크 에러, 서버 에러 등)
- **When** `createShareLink` 호출 실패
- **Then** 에러 토스트 "공유 링크 생성에 실패했습니다" (한국어, destructive variant) + Sentry `reportErrorToSentry` 호출

**AC-5 (도메인 핵심):** 토스트 메시지 한국어
- **Given** 앱 전체 언어가 한국어
- **When** 성공/실패 토스트 표시
- **Then** 모든 토스트 메시지가 한국어로 표시. 영어 메시지 0건

**AC-6 (접근성):** 키보드 내비게이션
- **Given** 공유 버튼에 Tab으로 포커스
- **When** Enter 키 입력
- **Then** `handleShare()` 실행. `aria-label="진단 결과 공유 링크 생성"` 스크린리더 안내

**AC-7 (성능):** 클립보드 복사 포함 전체 응답 ≤ 600ms
- **Given** CMD-SHARE-001 응답 ≤ 500ms (REQ-NF-006)
- **When** 버튼 클릭 → 링크 생성 → 클립보드 복사 → 토스트 표시
- **Then** 전체 사용자 체감 응답 ≤ 600ms (500ms API + ~100ms 클립보드+렌더링)

## 9. 🚧 Open Questions / Risks (보류 사항)

1. **iOS Safari 클립보드 권한:** iOS Safari는 사용자 제스처(click/tap) 컨텍스트 내에서만 `navigator.clipboard.writeText()` 허용. 현재 구현은 `handleShare()` 내 직접 호출이므로 제스처 컨텍스트 유지. 비동기 Server Action 응답 후 호출 시 제스처 컨텍스트가 소실될 수 있음 — 실기 테스트 필요.
2. **비HTTPS 환경:** 로컬 개발(http://localhost)에서 `navigator.clipboard`가 동작하지 않을 수 있음. fallback이 정상 동작하는지 확인 필요.
3. **토스트 자동 닫힘 시간:** 현재 shadcn/ui Toast 기본값 사용. URL이 긴 경우 토스트가 너무 빨리 닫힐 수 있음 — `duration` 옵션 커스터마이징 검토.
4. **동일 진단 중복 공유 링크:** CMD-SHARE-001에서 중복 생성 허용. UI에서 "이미 생성된 링크가 있습니다" 안내가 필요한지 — 현재는 매 클릭마다 새 링크 생성.
