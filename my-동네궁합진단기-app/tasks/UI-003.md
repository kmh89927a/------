## 1. 🎯 Summary

- **기능명:** [UI-003] 진단 결과 지도 시각화 UI — react-kakao-maps-sdk 교집합 후보 동네 마커 + 로딩 스켈레톤 + 에러 토스트
- **목적 (Why):**
  - **비즈니스:** 진단 핵심 화면. 교집합 후보 동네를 카카오맵 위에 시각화하여 직관적 결과 확인.
  - **사용자 가치:** 지도 기반 후보 동네 비교, 마커 클릭으로 상세 정보 접근, 로딩/에러 상태의 명확한 피드백.
- **범위 (What):**
  - ✅ 만드는 것: `components/diagnosis/MapCanvas.tsx` (Client — react-kakao-maps-sdk), `components/diagnosis/MarkerLayer.tsx`, `components/diagnosis/MapContext.tsx` (useSelectedCandidate/useFilteredCandidates 컨텍스트), 로딩 스켈레톤, 에러 토스트, EmptyState
  - ❌ 만들지 않는 것: 진단 로직(CMD-DIAG), 상세 패널(UI-004), 필터 UI(UI-005), 결제 UI
- **복잡도:** H
- **Wave:** 4

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 후보 동네 마커 ≥3개 렌더링
- **Given** 진단 결과에 `candidates.length >= 3` (MOCK 또는 실제)
- **When** `/diagnosis/result` 렌더링
- **Then** 카카오맵 위에 마커 ≥ 3개 표시, 각 마커 클릭 가능

**AC-2 (정상):** 로딩 스켈레톤 표시
- **Given** 진단 결과 로딩 중 (`isLoading === true`)
- **When** `/diagnosis/result` 접근
- **Then** shadcn Skeleton 컴포넌트 렌더링, 지도 영역에 반투명 오버레이

**AC-3 (예외):** 에러 토스트 — 타임아웃
- **Given** CMD-DIAG-006 타임아웃 에러 발생 (`TRANSPORT_API_TIMEOUT`)
- **When** 에러 상태 감지
- **Then** "교통 정보를 가져오지 못했습니다. 다시 시도해주세요." 토스트 (한국어), Sentry 로그

**AC-4 (예외):** EmptyState — 후보 0곳
- **Given** 진단 결과 `candidates.length === 0`
- **When** 렌더링
- **Then** "조건을 만족하는 동네가 없습니다" 안내 + 조건 완화 제안 ≥ 2개

**AC-5 (경계):** 카카오맵 SDK 키 미설정
- **Given** `NEXT_PUBLIC_KAKAO_MAP_KEY` 환경변수 미설정
- **When** 지도 로드 시도
- **Then** 카카오맵 대신 "지도를 불러올 수 없습니다. 관리자에게 문의하세요." 안내 표시

**AC-6 (UI/접근성):** 마커 클릭 → 상세 패널 연동 (UI-004)
- **Given** 마커 클릭
- **When** `useSelectedCandidate()` 컨텍스트 갱신
- **Then** `selected` 상태 업데이트, UI-004 상세 패널 열림 트리거

**AC-7 (UI):** 모바일 반응형
- **Given** viewport 375px
- **When** 지도 렌더링
- **Then** 지도 높이 `h-[60vh]`, 하단 카드 스크롤 가능
