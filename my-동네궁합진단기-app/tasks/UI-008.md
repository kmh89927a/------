## 1. 🎯 Summary

- **기능명:** [UI-008] 회원가입 유도 모달 UI
- **목적:** 비로그인 사용자가 무료 미리보기 1곳 소진 후 추가 동네 접근 시 회원가입 유도 모달 표시. 회원가입 CTA → UI-001 `/login` 리다이렉트.
- **범위:**
  - ✅ ConversionModal Client Component, shadcn/ui Dialog, CMD-SHARE-003 CTA 메시지, `/login` 리다이렉트, ESC/X 닫기
  - ❌ 결제 코드·타입·설정 일체, NextAuth.js, AES-256, PG 연동
- **복잡도:** M | **Wave:** 4

### ⚠️ 정합성 분기 결정

- **REQ-FUNC-013** (§4.1.2): "비회원이 무료 미리보기 1곳을 소진한 후 추가 동네 상세 조회를 시도하면, 유료 전환 유도 화면을 표시해야 한다. 전환 유도부터 결제 완료까지 단계 수는 3단계 이하여야 한다."
- **REQ-FUNC-014** (§4.1.2): "비회원이 무료 미리보기 소진 후 2곳째 동네 접근 시도 시, 유료 전환 유도 모달을 300ms 이내에 표시해야 한다. 뒤로가기 시 원래 리포트로 복귀하며, 강제 이탈을 방지해야 한다. 모달 노출 후 이탈률은 50% 이하를 목표로 모니터링해야 한다."
- **결정: 분기 C → 분기 B** (회원가입 유도 모달)
- **근거:** REQ-FUNC-013에 "결제 완료" 1건 존재하나 OS-09에서 결제 시스템 MVP Out-of-Scope. SRS 본문 모호성으로 회원가입 유도 모달로 해석.

## 3. 🛠️ Task Breakdown (실행 체크리스트)

- [ ] **3.1** `components/share/conversion-modal.tsx` — ConversionModal Client Component
  ```typescript
  'use client';
  import { Dialog, DialogContent, DialogTitle, DialogDescription } from '@/components/ui/dialog';
  import { Button } from '@/components/ui/button';
  import { useRouter } from 'next/navigation';
  import { UserPlus } from 'lucide-react';

  interface ConversionModalProps {
    open: boolean;
    onClose: () => void;
    ctaMessage: string;
  }

  export function ConversionModal({ open, onClose, ctaMessage }: ConversionModalProps) {
    const router = useRouter();
    return (
      <Dialog open={open} onOpenChange={(isOpen) => { if (!isOpen) onClose(); }}>
        <DialogContent id="conversion-modal" className="sm:max-w-md" aria-labelledby="conversion-title">
          <DialogTitle id="conversion-title">전체 후보 동네 정보를 보고 싶으세요?</DialogTitle>
          <DialogDescription>{ctaMessage}</DialogDescription>
          <div className="flex gap-2 mt-4">
            <Button id="signup-cta-button" onClick={() => router.push('/login')} className="flex-1">
              <UserPlus className="mr-2 h-4 w-4" />회원가입하기
            </Button>
            <Button id="dismiss-button" variant="outline" onClick={onClose} className="flex-1">나중에</Button>
          </div>
        </DialogContent>
      </Dialog>
    );
  }
  ```

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 잠금 후보 클릭 → 모달 표시 ≤300ms
- **Given** 비로그인 사용자가 UI-007 공유 리포트에서 블러 처리된 잠금 후보를 보는 상태
- **When** 잠금 후보 카드 클릭
- **Then** ConversionModal 표시 ≤300ms. 제목+CTA 메시지 포함

**AC-2 (정상):** 회원가입 버튼 → /login 리다이렉트
- **Given** ConversionModal 표시 상태
- **When** "회원가입하기" 클릭
- **Then** `router.push('/login')` → UI-001 이동

**AC-3 (정상):** ESC/X → 모달 닫힘 + 미리보기 복귀
- **Given** ConversionModal 표시 상태
- **When** ESC 또는 X 클릭
- **Then** 모달 닫힘, 무료 미리보기 복귀

**AC-4 (예외):** "나중에" → 모달 닫힘
- **Given** ConversionModal 표시 상태
- **When** "나중에" 클릭
- **Then** onClose 호출, 미리보기 복귀

**AC-5 (도메인 핵심 — 정합성):** 결제 키워드 0건 정적 검증
- **Given** `components/share/conversion-modal.tsx`
- **When** `grep -ri "결제\|Payment\|PG\|구독\|checkout"` 실행
- **Then** 매칭 0건

**AC-6 (접근성):** 키보드 내비게이션
- **Given** 모달 표시 상태
- **When** Tab 이동
- **Then** 회원가입→나중에→X 순서. Enter 실행. aria-labelledby 스크린리더 안내
