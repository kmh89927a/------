## 1. 🎯 Summary

- **기능명:** [UI-002] 주소 입력 화면 UI — 두 직장 주소 자동완성 필드 + 모드 선택(커플/싱글) + 진단 시작 버튼
- **목적 (Why):**
  - **비즈니스:** 진단 핵심 입력 화면. 두 직장 주소를 정확히 입력받아 교집합 후보 동네 진단의 시작점 역할.
  - **사용자 가치:** 자동완성으로 빠른 주소 입력(디바운스 300ms), 수도권 검증으로 비수도권 차단 안내, 모드 선택으로 커플/싱글 분기.
- **범위 (What):**
  - ✅ 만드는 것: `app/diagnosis/page.tsx` (Server), `components/diagnosis/AddressForm.tsx` (Client — react-hook-form + Zod), AddressInput 자동완성, 모드 선택 RadioGroup, 진단 시작 버튼 활성화 조건
  - ❌ 만들지 않는 것: 진단 로직(CMD-DIAG), Geocoding 백엔드(CMD-DIAG-001), 결제 UI
- **복잡도:** M
- **Wave:** 4

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 두 주소 입력 + 수도권 통과 → 진단 시작 활성화
- **Given** 수도권 주소 A, B 모두 자동완성 선택 완료
- **When** 수도권 검증 통과
- **Then** "진단 시작" 버튼 `disabled={false}`, 클릭 시 `/diagnosis/result` 리다이렉트

**AC-2 (예외):** 주소 1개만 입력 → 버튼 비활성화 + 인라인 에러
- **Given** addressA만 입력, addressB 비어있음
- **When** "진단 시작" 버튼 클릭 시도
- **Then** 버튼 `disabled={true}`, "두 번째 주소를 입력해 주세요" 인라인 에러 200ms 이내 표시

**AC-3 (예외):** 비수도권 주소 → 토스트 + 버튼 비활성화
- **Given** 주소 A에 "부산시 해운대구" 입력
- **When** 자동완성 결과 선택
- **Then** `isWithinSeoulMetropolitan` 실패 → "현재 수도권만 지원됩니다" 토스트 + 버튼 비활성화

**AC-4 (경계):** 자동완성 디바운스 300ms
- **Given** 주소 입력 필드에 빠르게 5자 연속 타이핑
- **When** 마지막 타이핑 후 300ms 경과
- **Then** `searchAddress` API 1회만 호출

**AC-5 (UI/접근성):** 모바일 반응형 + 키보드 내비게이션
- **Given** viewport 375px
- **When** `/diagnosis` 접근
- **Then** 입력 필드 `w-full`, 버튼 터치 영역 ≥ 44px, Tab으로 addressA → addressB → 모드 → 진단 시작 순 이동
