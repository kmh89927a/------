## 1. 🎯 Summary

- **기능명:** [UI-005] 조건 필터 UI — 통근 시간 슬라이더·예산 범위·필터 적용 시 실시간 지도 갱신
- **목적 (Why):**
  - **비즈니스:** 조건 필터로 후보 동네를 실시간 정제하여 탐색 효율 극대화.
  - **사용자 가치:** 슬라이더 조작 시 즉시(≤1초) 지도 마커가 갱신되어 직관적 탐색. 디바운스 200ms로 과도한 연산 방지.
- **범위 (What):**
  - ✅ 만드는 것: `components/diagnosis/FilterPanel.tsx` (Client — shadcn Slider + react-hook-form + Zod), `components/diagnosis/FilterResetButton.tsx`, UI-003 지도 갱신 연동 (useFilteredCandidates)
  - ❌ 만들지 않는 것: 필터 로직 자체(CMD-DIAG-005), 진단 로직(CMD-DIAG), 지도 자체(UI-003), 결제 UI

- **Wave:** 4

- [ ] **3.2** `components/diagnosis/FilterPanel.tsx` — `'use client'` + shadcn Slider
  ```typescript
  'use client';
  import { Slider } from '@/components/ui/slider';
  import { Label } from '@/components/ui/label';
  import { useFilteredCandidates } from '@/lib/diagnosis/use-filtered-candidates';

  export function FilterPanel({ candidates }: { candidates: CandidateAreaDTO[] }) {
    const { filtered, filters, updateFilters, filteredCount, totalCount } = useFilteredCandidates(candidates);

    return (
      <div className="space-y-6 px-4 py-4">
        {/* 통근 시간 슬라이더 */}
        <div className="space-y-2">
          <Label htmlFor="commute-slider">최대 통근 시간: {filters.maxCommuteTime ?? 60}분</Label>
          <Slider
            id="commute-slider"
            defaultValue={[60]}
            min={10}
            max={120}
            step={5}
            onValueChange={([value]) => updateFilters({ maxCommuteTime: value })}
            aria-label="최대 통근 시간 조절"
          />
          <div className="flex justify-between text-xs text-muted-foreground">
            <span>10분</span><span>120분</span>
          </div>
        </div>

        {/* 예산 범위 슬라이더 */}
        <div className="space-y-2">
          <Label htmlFor="budget-slider">예산 범위</Label>
          <Slider
            id="budget-slider"
            defaultValue={[3000, 15000]}
            min={1000}
            max={30000}
            step={500}
            onValueChange={([min, max]) => updateFilters({ budgetMin: min, budgetMax: max })}
            aria-label="예산 범위 조절"
          />
        </div>

        {/* 결과 카운트 */}
        <p className="text-sm text-muted-foreground">{filteredCount}/{totalCount}곳</p>
      </div>
    );
  }
  ```

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 슬라이더 변경 → 디바운스 200ms → 필터 적용 → 지도 갱신
- **Given** 10개 후보 표시, maxCommuteTime 슬라이더 60분
- **When** 슬라이더를 30분으로 변경
- **Then** 200ms 디바운스 후 `applyFilters` 실행, commuteA/B ≤ 30분 후보만 마커 표시

**AC-2 (경계):** p95 ≤ 1,000ms (REQ-NF-004)
- **Given** 100개 후보 데이터
- **When** 필터 슬라이더 변경
- **Then** 디바운스 200ms + 필터링 + 마커 갱신 총 ≤ 1,000ms

**AC-3 (예외):** 필터 결과 0개 → EmptyState
- **Given** maxCommuteTime: 10분 (모든 후보 초과)
- **When** 필터 적용
- **Then** `filteredCount === 0`, EmptyState 표시 + 조건 완화 제안 ≥ 2개

**AC-4 (정상):** 초기화 버튼 → 모든 필터 제거
- **Given** maxCommuteTime: 30분, budgetMax: 5000 필터 적용 상태
- **When** "필터 초기화" 버튼 클릭
- **Then** 모든 필터 제거, 전체 후보 마커 다시 표시

**AC-5 (UI/접근성):** 슬라이더 키보드 조작
- **Given** 통근 시간 슬라이더에 focus
- **When** 좌/우 화살표 키 입력
- **Then** 슬라이더 값 step(5분) 단위 변경, `aria-label` 존재

**AC-6 (UI):** 모바일 반응형
- **Given** viewport 375px
- **When** 필터 패널 표시
- **Then** 화면 상단 고정, 한 손 조작 가능, 스크롤 시 지도 아래에 슬라이더

## 9. 🚧 Open Questions / Risks (보류 사항)

1. **슬라이더 step 크기:** 통근 시간 5분 vs 10분 — UX 테스트 후 결정.
2. **예산 슬라이더 단위:** 월세(만원) vs 매매(억원) — 모드에 따라 동적 변경 검토.
3. **필터 상태 URL 동기화:** 필터 값을 URL query string에 반영하여 공유 가능하게 할지 — v1.5+.
4. **budgetMin/budgetMax 데이터 소스:** CandidateAreaDTO에 예산 필드 없음 — 매물 연동 시 추가 (CMD-DIAG-005 Open Question #4).
