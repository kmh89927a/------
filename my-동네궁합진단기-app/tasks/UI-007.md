## 1. 🎯 Summary

- **기능명:** [UI-007] SSR 공유 리포트 페이지 UI — 무료 미리보기 1곳 + 데이터 출처 배지 + OG 메타태그
- **목적 (Why):**
  - **비즈니스:** 배우자(비회원)가 카카오톡으로 받은 공유 링크를 클릭하면 앱 설치 없이 모바일 웹에서 진단 결과를 즉시 확인한다. 무료 미리보기 1곳으로 가치를 체험하고, 나머지 잠금 후보를 통해 회원가입 전환을 유도한다.
  - **사용자 가치:** 3G 환경에서도 2초 이내 로딩, 데이터 출처 배지로 정보 신뢰성 확인, 카톡 리치 프리뷰로 공유 시 미리보기 제공
- **범위 (What):**
  - ✅ 만드는 것: ShareReportView, PreviewCard, LockedCard(블러 처리), DataSourceBadge, SignupCTA(sticky), 모바일 우선 반응형, OG 메타태그(generateMetadata)
  - ❌ 만들지 않는 것: 전환 모달(UI-008), 동적 OG 이미지(/api/og), 결제 코드, NextAuth.js
- **복잡도:** H
- **Wave:** 4 (UI ShareLink 트랙)

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** SSR 렌더링 — 비로그인 + 유효 링크
- **Given** 유효한 ShareLink + 비로그인 사용자
- **When** `GET /share/{uuid}` 접근
- **Then** Server Component SSR 렌더링. 미리보기 1곳 전체 정보(출퇴근 시간, 스코어) + 나머지 후보 블러 처리(`filter: blur(8px)`) + SignupCTA sticky 하단 표시

**AC-2 (정상):** 로그인 사용자 → 전체 노출
- **Given** 유효한 ShareLink + 로그인 사용자
- **When** `GET /share/{uuid}` 접근
- **Then** 전체 후보 노출 (블러 0건), CTA 미표시, `lockedCandidates.length === 0`

**AC-3 (예외):** 만료 링크 → ExpiredView
- **Given** 만료된 ShareLink (`expiresAt < now`)
- **When** `GET /share/{uuid}` 접근
- **Then** `<ExpiredView />` 렌더링. 개인정보 노출 0건. "이 링크는 만료되었습니다" 메시지

**AC-4 (예외):** 비밀번호 보호 → PasswordPromptView
- **Given** `passwordHash`가 존재하는 ShareLink
- **When** `GET /share/{uuid}` 접근
- **Then** `<PasswordPromptView />` 렌더링. 진단 데이터 미노출

**AC-5 (정상):** OG 메타태그 검증
- **Given** 유효한 ShareLink
- **When** 카카오톡에서 URL 미리보기 요청 (generateMetadata 호출)
- **Then** `og:title`, `og:description`, `og:image` 포함. `twitter:card === 'summary_large_image'`. Lighthouse SEO 통과

**AC-6 (정상):** 데이터 출처 배지 표시 (REQ-FUNC-012)
- **Given** 진단 결과에 카카오 모빌리티, 국가공간정보포털 데이터 사용
- **When** ShareReportView 렌더링
- **Then** DataSourceBadge 2개 이상 표시. 출처명 + 최종 업데이트 일자 + trustLevel 포함. 출처 투명도 100%

**AC-7 (성능):** SSR 로딩 p95 ≤ 2,000ms (3G)
- **Given** 유효한 ShareLink
- **When** 3G throttling 환경에서 페이지 로딩
- **Then** Lighthouse 3G 측정 p95 ≤ 2,000ms

**AC-8 (접근성):** 비로그인 CTA → /login 리다이렉트
- **Given** 비로그인 사용자가 SignupCTA "회원가입하기" 클릭
- **When** 버튼 클릭
- **Then** `router.push('/login')` 실행 → UI-001 로그인 페이지로 리다이렉트

## 9. 🚧 Open Questions / Risks (보류 사항)

1. **`/api/og` 동적 OG 이미지:** `generateMetadata`에서 `images: ['/api/og?linkId=...']` 참조. 실제 OG 이미지 생성 Route Handler는 별도 태스크로 분리 필요. Next.js `ImageResponse` (Satori) 사용 예정.
2. **Edge Runtime 적용 여부:** Prisma가 Edge에서 제한적이므로 MVP에서는 Node.js runtime 유지.
3. **SEO noindex 여부:** 공유 링크 페이지를 검색엔진에 노출할지 — 바이럴 관점에서 index 허용이 유리.
4. **만료 페이지 HTTP 상태코드:** Next.js Server Component에서 HTTP 410 반환이 어려움. ExpiredView UI로 대체.
5. **PasswordPromptView 상태 관리:** 비밀번호 검증 성공 후 리로드 vs Client 상태 업데이트 — 현재 리로드 방식.
