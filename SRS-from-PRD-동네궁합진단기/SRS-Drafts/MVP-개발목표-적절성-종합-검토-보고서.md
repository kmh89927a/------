# MVP 개발 목표 적절성 종합 검토 보고서 (난이도/가능성/효율성)

**작성 목적:** "초급 개발 지식을 보유한 3년 차 IT 기획자가 AI를 활용해(엔지니어링 지식 투입 + Vibe-coding) MVP를 개발한다"는 가정하에, 현존하는 `SRS_v0.1_opus.md` 설계안을 시스템 및 비용 효율성 관점에서 점검하고 현실적인 대안을 제시합니다.

---

## 1. 개발 속도 관점 (MVP 단계의 과도한 시간 소요 방지)

### 🚨 현행 설계의 위험 요소
현재 SRS는 **MVP(최소 기능 제품)**라기보다는 상용화 1년 차 이후의 **강건한 프로덕션 시스템**에 가깝습니다. AI 기반 코딩 환경에서 단일 컨텍스트에 여러 도메인(인증, 크롤링, 외부 연계, 복잡한 알고리즘)이 맞물려 있으면, AI의 코드 결합도가 극도로 꼬이면서 **무한 디버깅 루프**에 빠져 개발 시간이 기하급수적으로 늘어납니다.

### 🛠️ 개발 속도 최적화를 위한 덜어내기 (Scope-Down)
1. **과도한 백그라운드 작업 폐기**
   - **데드라인 모드의 급매 매물 크롤링 (Cron Job):** 부동산 플랫폼 크롤링 방어 로직 우회 및 에러 핸들링은 AI 코딩만으로 구현하기 매우 어렵고 유지보수 비용이 막대합니다. 
   - *대안:* 매물 정보는 **네이버 부동산 등 외부 플랫폼 아웃링크(파라미터 조합 랜딩)** 방식으로 넘겨 개발 공수를 '0'으로 만드세요.
2. **연산 로직의 심플화**
   - **모드 분리 최소화:** MVP 단계에서는 '데드라인 모드', '싱글 모드'를 제거하고 오직 **'커플 동선 교차 진단' 단일 기능**에만 100% 매진해야 개발 기간 통제가 가능합니다.
3. **오프라인 파일럿 검증**
   - 사용자 데이터를 서버(DB)에 영구 저장하고 재탐색(Replay)하는 기능은 DB 스키마 복잡도를 높입니다. MVP에서는 브라우저의 `localStorage`를 이용해 클라이언트 단에만 과거 탐색 이력을 저장하는 방식으로 서버 개발 공수를 단축하세요.

## 2. 외부 연동 목표와 기술 스택 관점 (복잡성 및 리스크)

### 🚨 현행 설계의 위험 요소
* **기술 스택 (Next.js App Router + Server Actions):** 최신 프레임워크이나, 최신 AI 모델조차 V14/15의 클라이언트/서버 컴포넌트 경계 및 캐시(`revalidatePath` 등) 정책에 대해 잦은 환각(Hallucination) 에러를 발생시킵니다.
* **4중 폴백 전략 (카카오 → 네이버 → ODsay → DB):** 코드가 스파게티처럼 엉킬 확률 99%입니다.

### 🛠️ 연동 및 스택 최적화 방안
1. **단일 외부 API 활용 및 에러 처리 단순화**
   - 대중교통 API는 **오직 카카오 모빌리티 한 개**만 연동합니다. 
   - 장애 발생 시 "현재 교통 API 서비스 지연이 있습니다" 라는 안내창을 띄우는 것이 MVP에서는 가장 효율적인 방어벽입니다.
2. **인증 시스템 일원화**
   - `NextAuth.js` 대신 DB 환경으로 선택하신 **Supabase Auth**를 그대로 사용하는 것이 AI Vibe-coding 시 Next.js 구성과 DB 유저 관리를 통합할 수 있어 개발 속도가 2배 빠릅니다.
3. **공공데이터(경찰청 야간 치안 등) 연동 방식 변경**
   - 공공데이터포털 API의 느린 응답속도는 Vercel의 서버 타임아웃을 유발합니다. 
   - *대안:* API 실시간 연동이 아닌, 파이썬이나 오픈소스 툴로 1회 추출한 **정적 JSON 파일이나 앱 내 로컬 DB(SQLite/초보적 Supabase 테이블)**로 박아두고 매핑만 시키는 형태가 100배 효율적입니다.

## 3. 운영 소요 비용 관점 (클라우드 서비스 기반)

목표하신 "월 인프라 비용 무료 ~ 10만 원 이하"는 초기 트래픽 기준 충분히 가능합니다. 하지만 클라우드 컴퓨팅의 **제한 사항(Limits)**을 고려한 아키텍처 설계가 동반되어야 과금을 막고 서버가 뻗는 걸 방지할 수 있습니다.

### ☁️ 클라우드 효율성 및 비용 한계 돌파
1. **Vercel Serverless Function 타임아웃 방지**
   - Vercel Hobby(무료) 플랜은 API 함수 실행 시간이 **최대 10초**입니다. 프로($20/월)도 기본 설정은 제한적입니다. 
   - 만약 후보 동네 100곳의 교통 시간을 구하려 For-Loop 내에서 외부 API를 계속 호출한다면 10초를 무조건 넘깁니다. 
   - *대안:* 연산이 오래 걸리는 로직(여러 지점의 거리/시간 계산)은 서버에서 전체를 돌리지 말고, **클라이언트 사이드(브라우저)에서 병렬 호출(Promise.all)**로 넘기거나 계산 뎁스를 줄여 1~2초 이내에 반환되게 끔 구조를 바꿔야 추가 비용 결제를 막을 수 있습니다.
2. **Supabase 무료 티어 활용**
   - 프로젝트당 500MB DB, 월 2GB 대역폭이 무료입니다. MVP에서는 넘치도록 충분합니다. 다만 앞서 언급한 '캐시 DB, 4중 폴백 로그' 등을 쌓아두면 순식간에 트래픽과 용량을 갉아먹으니 폐기가 필수적입니다.
3. **외부 API (카카오 모빌리티) 요금**
   - 일 50만 건 무료 티어이므로 비용 발생 확률은 0%에 수렴합니다.
