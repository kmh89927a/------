## 1. 🎯 Summary

- **기능명:** [UI-010] 급매 매물 리스트 + 지도 동시 표시 UI — 0건 시 조건 완화 제안 UI + 알림 구독 옵션
- **목적:** 데드라인 모드에서 교집합 후보 동네를 지도+리스트로 동시 표시. 매물 카드 클릭 시 CMD-DL-002 네이버 아웃링크. 0건 시 CMD-DL-003 조건 완화 제안 + 알림 구독 UI 제공.
- **범위:**
  - ✅ 지도+리스트 동시 표시 (모바일: 상하 분할 또는 탭 전환, 데스크탑: 좌우 분할), UI-003 MapCanvas 활용, UI-009 TimelineCard import, CMD-DL-002 `openNaverListing` 호출, CMD-DL-003 `suggestRelaxedFilters`/`subscribeToListingAlerts` UI, EmptyState 컴포넌트, 모바일 우선 반응형, Pretendard 폰트
  - ❌ 자체 매물 데이터 로직(네이버 아웃링크만), 결제, NextAuth.js
- **복잡도:** M | **Wave:** 4 (UI-Deadline 트랙)

            onClick={() => onCardClick(listing)}
            role="listitem"
            tabIndex={0}

              </div>
              <ExternalLink className="w-4 h-4 text-muted-foreground" />
            </CardContent>

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 매물 N개 시 지도 + 리스트 동시 표시
- **Given** QRY-DL-001 `filterListings` 결과 `listings.length === 5`
- **When** 페이지 렌더링
- **Then** MapCanvas에 후보 마커 표시 + ListingCards에 5개 카드 표시. 동시 표시

**AC-2 (정상):** 카드 클릭 → 네이버 새 창 열기 (noopener,noreferrer)
- **Given** 매물 카드 "성남시 분당구" 표시
- **When** 카드 클릭
- **Then** `openNaverListing` 호출 → `window.open(url, '_blank', 'noopener,noreferrer')`. 네이버 부동산 새 탭

**AC-3 (정상):** 0건 시 EmptyState + 조건 완화 제안 + 알림 구독
- **Given** `listings.length === 0`
- **When** 페이지 렌더링
- **Then** EmptyState 표시: "조건에 맞는 급매 매물이 없습니다" + RelaxedFilterSuggestions(≥3개) + AlertSubscriptionForm

**AC-4 (도메인 핵심):** 자체 매물 데이터 로직 0건 (정적 검증)
- **Given** UI-010 전체 코드
- **When** `grep -ri "prisma.listing\|fetch.*listing\|model Listing" components/deadline/`
- **Then** 0건. 네이버 아웃링크만 사용 (CMD-DL-002)

**AC-5 (경계):** 모바일 vs 데스크탑 레이아웃 분기
- **Given** 모바일 (width < 768px)
- **When** 렌더링
- **Then** 상단 지도 (40vh) + 하단 리스트 (스크롤). 데스크탑: 좌측 리스트 (40%) + 우측 지도 (60%)

**AC-6 (예외):** 알림 구독 성공 시 확인 피드백

**AC-7 (도메인 핵심):** MapCanvas + TimelineCard import 정상

## 9. 🚧 Open Questions / Risks

1. **모바일 탭 vs 동시 표시 트레이드오프:** 현재 상하 분할 (상단 40vh 지도 + 하단 스크롤 리스트). 탭 전환 방식도 가능하나, 지도와 리스트 동시 확인이 사용자 의사결정에 더 유리. A/B 테스트 후 확정.
2. **MapCanvas 마커 클릭과 카드 동기화:** 지도 마커 클릭 시 하단 리스트 해당 카드로 스크롤. 구현 복잡도에 따라 MVP에서 제외 가능.
3. **자체 매물 데이터 절대 금지:** REQ-FUNC-016 — 네이버 부동산 아웃링크만. UI-010에서 자체 매물 fetch/조회 로직 작성 시 v1.3 정합성 위반.
4. **CMD-DL-003 조건 완화 → 재조회:** 조건 완화 "적용" 클릭 시 QRY-DL-001 `filterListings` 재호출 필요. 새 필터로 재조회 후 리스트 갱신.
