## Ⅰ. 문제 정의 및 고객별 핵심 문제 서술 (Pain & Needs)

### 1-0. 문제 정의 — 3가지 관점 통합

본 서비스가 해결하고자 하는 문제는 산업 분석(Porter 5 Forces, Value Chain)과 시장 리서치에서 도출된 세 가지 관점으로 정의됩니다.

#### 관점 1: 주거 정보 비대칭과 거주 미스매치 (프롭테크 측면)

프롭테크 산업에서 대형 플랫폼(네이버부동산, 직방 등)은 방대한 정량적 매물 데이터(가격·평수)를 제공하지만, 개인의 라이프스타일 맥락을 반영한 **정성적 체감 데이터는 심각하게 결핍**됩니다. 핵심 가치 창출 구조는 '공공 및 지리 데이터의 디지털 자산화', '초개인화 라이프스타일 피로도 스코어링 알고리즘 내재화', '기존 대형 플랫폼으로의 비적대적 리드 연결'입니다.

> **해결 과제:** 수천만 원의 보증금이 걸린 이주를 준비하는 1인 가구·맞벌이 부부가, 단편적 매물 정보만으로 실제 생활 패턴(오르막길 경사, 야간 소음, 출퇴근 체감 피로도 등)에 맞는 동네를 결정해야 하는 상황에서 발생하는 **주거 미스매치와 경제적 손실**.

#### 관점 2: 물리적 임장·탐색 생태계의 비효율 (하이퍼로컬 측면)

당근마켓 같은 하이퍼로컬 신뢰망이나 삼삼엠투·리브애니웨어 방식의 '미리 살아보기(단기 체류)'를 통해 동네의 실제 인프라를 검증하려는 수요가 폭발적으로 증가하고 있으나, 이 방식은 자본과 시간이 막대하게 소모됩니다.

> **해결 과제:** 시간과 금전 여유가 부족한 현대인이, 동네의 치안·거주 분위기를 검증하기 위해 오프라인 발품을 팔거나 단기 체류를 해야 하는 상황에서 발생하는 **신체적 피로 누적과 탐색 비용 낭비**.

#### 관점 3: 의사결정의 심리적 고립감과 정보 신뢰 부재 (초개인화 큐레이션 측면)

개인의 고유한 성향(MBTI, 반려견 여부, 혼밥 인프라 등)을 반영하여 동네 자체를 선택하고자 하는 초개인화 니즈가 진화하고 있으나, 취향 기반 B2C 대체 서비스의 경쟁도는 아직 블루오션에 가까운 반면 유튜브 '동네 브이로그' 등 대체재의 위협이 존재합니다.

> **해결 과제:** 모든 것을 스스로 결정해야 하는 1인 가구·예비 이사자가, 내 맥락을 객관적으로 증명해 줄 조율형 도구 없이 겪는 **짙은 심리적 불안감과 고립감, 결정 두려움**.

### 1-2. 세그먼트별 핵심 Pain & Needs

페르소나 스펙트럼 및 고객 여정지도(CJM) 분석, 14건 JTBD 인터뷰에서 도출된 세그먼트별 핵심 Pain을 정리합니다.

#### C-01 | 맞벌이 3040 부부형 — 두 동선 충돌의 의사결정 고통

**핵심 Pain:**

부부 두 사람의 출퇴근 동선을 동시에 만족하는 동네를 찾을 도구가 시장에 존재하지 않음. 각자 검색 후 수동 취합에 평균 4.2개월·임장 12회 소요. 정보가 분산된 앱들(네이버·직방·호갱노노) 사이에서 합의점을 찾지 못해 계약이 지연되거나 한 사람이 일방적으로 희생.

**부가 Needs:** 배우자 설득용 객관 비교 리포트 / 초품아 + 학원가 + 광역버스 착석 통합 화면 / 양가 부모 설득 자료

**CJM 핵심 Pain Point (①김지영 기준):**

| 여정 단계 | Pain Point | 개선 기회 |
| --- | --- | --- |
| 문제인식 | 두 직장 동선을 동시에 계산해 교집합 동네를 보여주는 도구 없음 | "두 직장 주소 입력 → 교차 동선 지도 즉시 시각화" 온보딩 훅 |
| 탐색 | 학군(호갱노노)·교통(카카오맵)·매물(직방)이 세 앱에 분산 | 학군 + 교통 + 매물 단일 화면 통합 |
| 발견·결제 | 의사결정 단위가 부부라 혼자 납득해도 배우자 동의 없이 결제 불가 | "배우자 공유 링크" + 무료 1곳 미리보기 |

| 유지 | 재방문 트리거 없음. 이전 입력값 저장 안 됨 | "2년 후 재진단 알림" + 이전 조건 자동 저장 |

#### C-02 | 맹모삼천지교형 — 학군·교통 복합 데이터 비효율

**핵심 Pain:**

대치동 학군을 목표로 하면서 부부 출퇴근 광역버스 착석 가능 여부를 동시에 검증할 디지털 수단이 전혀 없음. '착석 가능 노선'은 직접 탑승 실측 외 방법 無(평균 9회 실측). 학원 셔틀 종점 데이터는 디지털화 자체가 미비해 카페에서 수동 수집 후 지도에 핀을 직접 찍는 방식.

**부가 Needs:** 학원 셔틀 종점 DB / 중립 데이터 기반 스코어(중개사 편향 불신) / 학교 배정 구역 레이어

**CJM 핵심 Pain Point (②박상민 기준):**

| 여정 단계 | Pain Point | 개선 기회 |
| --- | --- | --- |
| 문제인식 | 부부 출퇴근과 학원가 접근성을 동시 계산하는 도구 없음. 엑셀 수동 정리에 수십 시간 | 부부 직장 + 학원가 반경 교차 필터링 전용 입력 폼 |
| 탐색 | 광역버스 착석 가능 여부·환승 횟수 같은 정보는 직접 임장 외 확인 불가 | 대중교통 환승 횟수·착석률 데이터 연동 |
| 발견·결제 | 중개사마다 추천이 달라 객관적 제3자 데이터 기반 스코어 부재 | 공공 실거래가·교통 데이터 기반 중립 스코어 |
| 사용 | 사립 학원 셔틀버스 정류장 데이터 미제공 | 학원 연합회 API 연동 셔틀 정류장 좌표 DB 구축 |
| 유지 | 예측 vs 실제 괴리에 대한 피드백 채널 없음 | 실거주 후기 수집 → 알고리즘 보정 데이터 피드백 루프 |

#### C-03 | 긴급 이사형 — D-2개월 시간 압박 속 조건 충족 불가

**핵심 Pain:**

집주인 통보·발령이 동시에 발생해 기존 탐색 기준이 전부 무효화. 직방·피터팬·네이버 부동산 3앱을 동시에 열어도 두 조건 교집합 매물을 한 번에 볼 수 없어 중복 확인 반복. 오늘 올라온 급매가 4.3시간 지연 후 발견되어 이미 소진된 경우 다수. 긴급 이사 세그먼트는 전체 이사 수요의 약 30%로 추산.

**부가 Needs:** 계약 후 체크리스트·비상 가이드(Anxiety 해소) / 오늘 급매 실시간 연동 / 30분 핵심 요약 패키지

**CJM 핵심 Pain Point (④정우진 기준):**

| 여정 단계 | Pain Point | 개선 기회 |
| --- | --- | --- |
| 문제인식 | 이사 기한(D-2개월) + 직장·거주지 동시 변화가 겹쳐 기존 탐색 방식 전부 무용지물 | "데드라인 모드" → 계약 가능 기한 역산 + 급매 매물 우선 필터링 |
| 탐색 | 새 직장 동선 + 아이 유치원 배정 구역 동시 만족 매물 한 번에 탐색 불가 | 직장 주소 + 배정 구역 동시 입력 → 교집합 즉시 추출 |
| 사용 | 계약 후 추가 변수(계약 파기·하자)에 대응하는 가이드 없음 | 계약 후 체크리스트 + 비상 대응 가이드 콘텐츠 |

#### C-04 | 반복 이사형 — 2~3년 주기 원점 재시작의 피로 누적

**핵심 Pain:**

교사·군인·공무원 등 순환 발령 구조로 2~3년마다 이사 반복. 매번 이전 탐색 과정을 기억하지 못해 원점 재시작(평균 3주). 미래 발령 변수를 현재 계약 결정에 반영하는 도구가 없어 '2년 후 이 집이 괜찮을까'는 항상 미해결 불안으로 남음. 관사 vs 전세 비용 비교도 수동 엑셀로 처리.

**부가 Needs:** 입력값 자동 저장 + 원클릭 재탐색 / 발령 시나리오 복수 입력 / 두 도시 간 거점 교차(지방 커버리지)

**CJM 핵심 Pain Point (⑤이수현 기준):**

| 여정 단계 | Pain Point | 개선 기회 |
| --- | --- | --- |
| 문제인식 | 2년마다 반복되는데 이전 탐색 과정 저장 안 됨 → 매번 원점 재시작 | 이전 입력값·결과 자동 저장 → "지난번 조건으로 재탐색" 원클릭 |
| 의사결정 | 2년 후 발령 변화 시나리오까지 반영하는 미래 시뮬레이션 없음 | "미래 발령 시나리오 비교" — 후보 지역 2~3곳 입력 시 동선 변화 미리보기 |
| 유지 | 발령 시기를 서비스가 모르니 재방문 트리거 없음 | 발령 주기 알림 설정 + 이전 입력값 자동 저장 |

#### A-01 | 이직 후 이사형 (Adjacent) — 학군 없는 싱글 이사의 도구 부재

**핵심 Pain:**

이직 확정 후 새 직장 + 여가 거점(홍대·합정 등) 두 조건의 교집합을 탐색할 때, 기존 서비스는 학군·가족 중심 항목이 결과를 오염시킴. 1인 가구 여성·싱글은 야간 귀가 치안·자전거 도로 등 라이프스타일 레이어가 핵심인데 이를 제공하는 서비스 無. '혼자 결정해야 한다'는 심리적 고립감도 채택 장벽.

**AOS vs DOS — 달라진 순위에서 읽히는 것:**

- **학원 셔틀 DB:** AOS 공동 1위(4.00) → DOS 8위(1.60)로 하락. 불만족 극심하나 데이터 확보 난이도가 너무 높아, 로드맵 후순위 판단이 수치로 뒷받침됨.
- **배우자 공유 링크:** AOS 6위 → DOS 3위(3.65)로 상승. 구현 단순하고 결제 전환·바이럴 모두 직결. 가장 빠르게 실행해야 할 기능.
- **추천 리워드:** DOS 유일 음수(-0.20). 만족도가 중요도보다 높아 자발적 추천이 이미 발생 중. 공식 리워드에 자원 투입 불필요.
