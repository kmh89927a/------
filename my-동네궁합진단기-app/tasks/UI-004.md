## 1. 🎯 Summary

- **기능명:** [UI-004] 후보 동네 상세 정보 패널 UI — 출퇴근 시간·교통수단·환승횟수 표시 + 출근 시간대 변경 컨트롤
- **목적 (Why):**
  - **비즈니스:** 마커 클릭 시 후보 동네의 통근 상세 정보를 제공하여 의사결정 근거 강화.
  - **사용자 가치:** 양쪽 직장까지 출퇴근 시간·교통수단·환승 정보를 한눈에 확인하고, 출근 시간대를 변경하여 시뮬레이션.
- **범위 (What):**
  - ✅ 만드는 것: `components/diagnosis/CandidateDetailPanel.tsx` (Client), `components/diagnosis/CommuteInfo.tsx`, `components/diagnosis/TimeSlotSelector.tsx`, 모바일 Sheet + 데스크탑 사이드 패널
  - ❌ 만들지 않는 것: 진단 로직, 지도 자체(UI-003), 필터 UI(UI-005), 결제 UI
- **복잡도:** M
- **Wave:** 4

  'use client';
  import { useSelectedCandidate } from '@/components/diagnosis/MapContext';
  import { Sheet, SheetContent } from '@/components/ui/sheet';

  export function CandidateDetailPanel() {
    const { selected, close } = useSelectedCandidate();
    if (!selected) return null;
    return (
      <Sheet open={!!selected} onOpenChange={(open) => !open && close()}>
        <SheetContent side="bottom" className="md:side-right md:w-[400px]">
          <h2 className="text-lg font-bold">{selected.name}</h2>

          <CommuteInfo info={selected.commuteB} label="직장 B까지" />

        </SheetContent>
      </Sheet>
    );
  }

## 4. ✅ Acceptance Criteria (GWT 패턴, BDD)

**AC-1 (정상):** 마커 클릭 → 상세 패널 열림
- **Given** 지도에 후보 동네 마커 표시
- **When** 마커 클릭
- **Then** `useSelectedCandidate()` 갱신 → 상세 패널 열림, 동네 이름·출퇴근 정보 표시

**AC-2 (정상):** 출퇴근 시간·교통수단·환승 표시
- **Given** 상세 패널 열린 상태
- **When** 렌더링
- **Then** 직장 A까지: `durationMinutes`분, 교통수단 아이콘, 환승 `transfers`회 표시. 직장 B도 동일

**AC-3 (예외):** 출근 시간대 변경 → 데이터 갱신
- **Given** 시간대 "08:00" 선택 상태
- **When** "09:00" 탭 클릭
- **Then** QRY-DIAG-002 재호출 (또는 캐시), 출퇴근 시간 값 갱신. 응답 p95 ≤ 2,000ms

**AC-4 (경계):** 닫기 — 버튼 또는 외부 클릭
- **Given** 상세 패널 열린 상태
- **When** 닫기 버튼 클릭 또는 Sheet 외부 클릭 또는 Escape 키
- **Then** 패널 닫힘, `selected === null`

**AC-5 (UI/접근성):** 모바일 Sheet 슬라이드 업
- **Given** viewport 375px (모바일)
- **When** 마커 클릭
- **Then** 화면 하단에서 Sheet 슬라이드 업, 스와이프 다운으로 닫기 가능

**AC-6 (UI):** 데스크탑 사이드 패널
- **Given** viewport 1280px (데스크탑)
- **When** 마커 클릭
- **Then** 오른쪽 사이드 패널(400px 너비) 열림, 지도와 동시 표시

## 9. 🚧 Open Questions / Risks (보류 사항)

1. **출근 시간대 변경 시 캐시 vs 재호출:** 캐시 우선으로 구현하되, 캐시 미스 시 QRY-DIAG-002 재호출. 캐시 TTL은 세션 기간.
2. **Sheet side 반응형:** shadcn Sheet의 `side` prop이 반응형으로 동적 변경 가능한지 — CSS `@media` 또는 `useMediaQuery` 훅으로 분기.
3. **자차 모드 데이터:** 카카오 모빌리티 API의 자차 경로 데이터 가용성 — transit 우선 표시, 자차는 v1.5+.
4. **스와이프 제스처:** 모바일 Sheet 스와이프 다운 닫기 — Radix UI Sheet 기본 동작으로 커버 가능한지 확인.
